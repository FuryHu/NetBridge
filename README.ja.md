# NetBridge

[English](README.md) | [简体中文](README.zh-CN.md) | **日本語**

インターネット越しに LAN 専用ゲーム（や任意の LAN ソフト）を友達と遊ぶための軽量な仮想 LAN ツール。UDP 上にプライベート仮想ネットワークを構築し、NAT 越えによる直接 P2P 接続を優先、ホールパンチ失敗時はサーバー中継にフォールバックします。

> 「友達がルーターの裏にいて、ゲームの LAN ロビーが空っぽ」という古典的な問題（例：『ウォークラフト III』『レッドアラート2』『エイジ・オブ・エンパイア』『スタークラフト』『文明 VI』などの LAN マルチ）のために作りました。

## ✨ 特徴

- **仮想 LAN** - 各ルームに `10.66.0.0/16` のサブネットを割り当て、peer は安定した仮想 IP（再起動後も再利用）を持ち、同じスイッチに繋がっているように見えます。
- **P2P 優先・中継フォールバック** - IPv4 と IPv6 両方に並行で UDP ホールパンチを行い、3 秒で繋がらなければサーバー中継に切り替えます。UI は各 peer のチャネル（P2P / 中継）とプロトコルファミリ（IPv4/IPv6）を表示します。
- **LAN 発見に優しい** - ブロードキャスト/マルチキャストをルーム全員に配信するため、ゲームロビーのスキャンがそのまま通じます。
- **ルームチャットと音声** - 同じ P2P/中継チャネル上でテキストチャットと音声通話。WebCodecs による Opus（旧版 WebView2 は生 PCM にフォールバック）、エコーキャンセル、peer ごとの音量・ミュート。
- **多言語 UI** - 簡体字中国語・英語・日本語の 3 言語。トップバーから切替可能（起動を跨いで保持）。
- **シングルバイナリサーバー** - 純 Go、ランタイム依存なし。小さな Linux VPS 上で systemd 配下で動きます。

## 🏗️ アーキテクチャ

3 つのモジュールからなる Go ワークスペースです：

| モジュール | 役割 |
| --- | --- |
| `protocol/` | 共有線プロトコル - JSON シグナリング + コンパクトなバイナリデータフレーム、コーデック、定数。 |
| `server/` | UDP シグナリング + 中継サーバー。VIP 割り当て、ルーム/peer 管理、ホールパンチ補助のためのエンドポイント配信、P2P 失敗時の中継。純 Go。 |
| `client/` | デスクトップアプリ（[Wails v2](https://wails.io) = Go + Vue 3）。Windows では [wintun](https://www.wintun.net/) で NIC を管理し、TUN とネットワーク層の間で IP パケットをブリッジします。 |

```
        ┌───────────────┐         UDP :10555         ┌───────────────┐
        │   Client A    │ ◄════ signaling / relay ═══►│   Client B    │
        │  (Wails+TUN)  │                            │  (Wails+TUN)  │
        └───────┬───────┘                            └───────┬───────┘
                │ hole-punch (v4 & v6 in parallel)          │
                └══════════════ P2P game data ══════════════┘
                          (falls back to server relay)
                              ▲
                              │
                       ┌──────┴──────┐
                       │   Server    │
                       │  (pure Go)  │
                       └─────────────┘
```

## 🧱 技術スタック

- **Go** 1.26 · **Wails** v2.12 · **Vue 3** + **Vite** 3
- **wintun**（Windows 仮想 NIC）· **google/uuid**
- サーバーは共有 `protocol` モジュール以外にサードパーティ依存を持ちません。

## 🚀 クイックスタート

### サーバー

UDP `10555` を開けた VPS にデプロイします：

```bash
cd server
go run .                                  # デフォルト: [::]:10555 で待受（デュアルスタック）
go build -o netbridge-server . && ./netbridge-server   # またはビルドして実行
```

任意のフラグ / 環境変数：

| Flag | Env | デフォルト | 説明 |
| --- | --- | --- | --- |
| `-addr` | - | `[::]:<port>` | 待受アドレス（例: `0.0.0.0:10555`、v4 のみ） |
| `-port` | `NETBRIDGE_PORT` | `10555` | UDP 待受ポート |
| `-room-timeout` | `NETBRIDGE_ROOM_TIMEOUT` | `15` | peer オフラインタイムアウト（秒） |

ログは構造化テキストで stdout に出力されます（systemd 配下では journald が取得）。

### クライアント（Windows）

[Wails CLI](https://wails.io/docs/gettingstarted/installation) と Go が必要です：

```bash
cd client
wails dev      # ライブ開発（ホットリロード）
wails build    # 再配布可能な .exe / NSIS インストーラを生成
```

> wintun NIC の作成には**管理者権限**が必要です - マニフェストで `requireAdministrator` を宣言済み。`wails dev` には `RestartAsAdmin` のフォールバックがあります。

使い方：起動 -> サーバーアドレス（`vps-ip:10555`）を入力 -> ニックネームでルームに参加 -> ルーム名を共有。サーバーが VIP を割り当てると仮想 NIC が自動で立ち上がります。

## ⚙️ 仕組み

サーバーが VIP を割り当て、全員の公開 v4/v6 エンドポイントを配信します。クライアントは両経路に並行でホールパンチを行い、最初に繋がった方が P2P チャネルになります。3 秒以内にどちらも繋がらなければサーバーが中継します（宛先 VIP でルーティングし、ペイロードは覗きません）。TUN ブリッジが仮想 NIC と P2P/中継チャネルの間で生 IP パケットを搬送します。

主要定数（`protocol/constants.go`）：サブネット `10.66.0.0/16`、MTU `1400`、ハートビート `5s`、peer タイムアウト `15s`、ホールパンチタイムアウト `3s`、ポート `10555`。

## 📝 メモ

- クライアントは Windows 専用（wintun）。サーバーは Go が動く環境ならどこでも動きます。
- サーバーはシグナリング・中継ともに UDP `10555` を開けてください。
- P2P の成否は NAT タイプに依存します。対称型 NAT は通常中継にフォールバックします。
