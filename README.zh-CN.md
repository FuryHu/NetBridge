# NetBridge

[English](README.md) | **简体中文** | [日本語](README.ja.md)

一个轻量级虚拟局域网工具，让你和朋友跨越互联网玩局域网联机游戏（或使用任何局域网软件）。它在 UDP 之上构建一张私有虚拟网络，优先用 NAT 穿透建立直连（P2P），打洞失败时回落到服务器中转。

> 专门解决经典的「朋友在路由器后面、游戏局域网大厅空空如也」问题--例如《魔兽争霸 III》《红色警戒 2》《帝国时代》《星际争霸》《文明 6》等局域网联机。

## ✨ 特性

- **虚拟局域网**--每个房间分配 `10.66.0.0/16` 网段，成员拿到稳定虚拟 IP（跨重启复用），彼此如同接在同一台交换机。
- **P2P 优先、中转兜底**--并行向 IPv4 与 IPv6 打洞，3 秒未通则回落服务器中转。界面显示每个对端的通道（P2P / 中转）与协议族（IPv4/IPv6）。
- **局域网发现友好**--广播/多播分发给房间内全部成员，游戏大厅扫描开箱即用。
- **房间聊天与语音**--同一条 P2P/中转通道上的文字聊天与语音。WebCodecs Opus（旧版 WebView2 降级裸 PCM）、回声消除、单端音量与静音。
- **界面多语言**--简体中文、English、日本語三语，顶栏切换（跨启动持久化）。
- **单文件服务端**--纯 Go，无运行时依赖，小 Linux VPS + systemd 即可。

## 🏗️ 架构

一个包含三个模块的 Go 工作区：

| 模块 | 职责 |
| --- | --- |
| `protocol/` | 共享线协议--JSON 信令 + 紧凑二进制数据帧、编解码、常量。 |
| `server/` | UDP 信令 + 中转服务端。分配 VIP、维护房间与成员、广播端点协助打洞、P2P 失败时中转。纯 Go。 |
| `client/` | 桌面客户端（[Wails v2](https://wails.io) = Go + Vue 3）。Windows 上通过 [wintun](https://www.wintun.net/) 管理网卡，在 TUN 与网络层之间桥接 IP 包。 |

```
        ┌───────────────┐         UDP :10555         ┌───────────────┐
        │   客户端 A    │ ◄════ 信令 / 中转 ═════════►│   客户端 B    │
        │ (Wails + TUN) │                            │ (Wails + TUN) │
        └───────┬───────┘                            └───────┬───────┘
                │ 打洞（v4 与 v6 并行）                      │
                └══════════════ P2P 游戏数据 ═══════════════┘
                       （失败时回落到服务器中转）
                              ▲
                              │
                       ┌──────┴──────┐
                       │   服务端    │
                       │  （纯 Go）  │
                       └─────────────┘
```

## 🧱 技术栈

- **Go** 1.26 · **Wails** v2.12 · **Vue 3** + **Vite** 3
- **wintun**（Windows 虚拟网卡）· **google/uuid**
- 服务端除共享 `protocol` 模块外无任何第三方依赖。

## 🚀 快速开始

### 服务端

部署到开放了 UDP `10555` 的 VPS：

```bash
cd server
go run .                                  # 默认监听 [::]:10555（双栈）
go build -o netbridge-server . && ./netbridge-server   # 或构建后运行
```

可选参数 / 环境变量：

| 参数 | 环境变量 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `-addr` | - | `[::]:<port>` | 监听地址，如 `0.0.0.0:10555`（仅 v4） |
| `-port` | `NETBRIDGE_PORT` | `10555` | UDP 监听端口 |
| `-room-timeout` | `NETBRIDGE_ROOM_TIMEOUT` | `15` | peer 离线超时（秒） |

日志以结构化文本输出到 stdout（systemd 下由 journald 接管）。

### 客户端（Windows）

需要 [Wails CLI](https://wails.io/docs/gettingstarted/installation) 与 Go：

```bash
cd client
wails dev      # 实时开发（热重载）
wails build    # 生成可分发的 .exe / NSIS 安装包
```

> 创建 wintun 虚拟网卡需要**管理员权限**--清单已声明 `requireAdministrator`；`wails dev` 下有 `RestartAsAdmin` 兜底。

使用方式：启动 -> 填服务端地址（`vps-ip:10555`）-> 用昵称加入房间 -> 把房间号告诉朋友。服务端分配 VIP 后虚拟网卡自动开启。

## ⚙️ 工作原理

服务端分配 VIP 并广播所有人的公网 v4/v6 端点。客户端并行向两条路径打洞，先通者成为 P2P 通道；3 秒内都不通则由服务端中转（按目的 VIP 路由，不解析负载）。TUN 桥在虚拟网卡与 P2P/中转通道之间搬运裸 IP 包。

关键常量（`protocol/constants.go`）：网段 `10.66.0.0/16`、MTU `1400`、心跳 `5s`、peer 超时 `15s`、打洞超时 `3s`、端口 `10555`。

## 📝 说明

- 客户端仅支持 Windows（wintun）；服务端可在任何能跑 Go 的平台运行。
- 服务端需开放 UDP `10555`，信令与中转都走它。
- P2P 能否成功取决于 NAT 类型；对称型 NAT 通常回落到中转。
