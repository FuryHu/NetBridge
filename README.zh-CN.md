# NetBridge

[English](README.md) | **简体中文**

一个轻量级虚拟局域网工具，让你和朋友跨越互联网玩局域网联机游戏（或使用任何局域网软件）。它在 UDP 之上构建一张私有虚拟网络，优先用 NAT 穿透建立直连（P2P），打洞失败时自动回落到服务器中转。

> 专门解决经典的"朋友在路由器后面、游戏局域网大厅空空如也"问题——例如《文明 6》的局域网联机。

## ✨ 特性

- **虚拟局域网**——每个房间分配 `10.66.0.0/16` 网段，成员按加入顺序拿到虚拟 IP（`10.66.0.2`、`10.66.0.3`……），彼此看起来就像接在同一台交换机上。
- **P2P 优先、中转兜底**——UDP 打洞采用双栈策略：并行向 IPv4 与 IPv6 端点打洞，谁先通走谁。打洞超时（3 秒）后流量透明地经由服务器中转。
- **虚拟 IP 稳定**——每台机器持有持久化的 PeerID，服务端跨重启、跨重进房间都复用同一个 VIP，"你的 IP"不会漂移。
- **局域网发现友好**——受限/子网广播（`255.255.255.255`、`10.66.255.255`）与多播（`224.0.0.0/4`）都会分发给房间内全部成员，游戏大厅扫描"开箱即用"。
- **房间聊天**——房内文字聊天，由服务端广播给所有人。
- **延迟可见**——基于心跳的保活 + RTT 测量，界面显示每个对端的通道（P2P / 中转 / 无）与协议族（IPv4/IPv6）。
- **紧凑二进制数据帧**——信令继续走 JSON 方便调试，但高吞吐数据通道用 12 字节二进制头封装，避免 JSON+base64 带来的约 35% 膨胀撑爆 1400 MTU、引发分片丢包。
- **单文件服务端**——纯 Go，无运行时依赖，跑在一台小 Linux VPS 上配合 systemd 即可。

## 🏗️ 架构

NetBridge 是一个包含三个模块的 Go 工作区：

| 模块 | 职责 |
| --- | --- |
| `protocol/` | 共享线协议——JSON 信令报文 + 紧凑二进制数据帧、编解码、常量。 |
| `server/` | UDP 信令 + 中转服务端。分配虚拟 IP、维护房间与成员、广播对端端点协助打洞、P2P 失败时中转数据。纯 Go。 |
| `client/` | 桌面客户端（[Wails v2](https://wails.io) = Go + Vue 3）。在 Windows 上通过 [wintun](https://www.wintun.net/) 管理虚拟网卡，在 TUN 设备与网络层之间桥接 IP 包，并驱动界面。 |

```
        ┌───────────────┐         UDP :10555         ┌───────────────┐
        │   客户端 A    │ ◄════ 信令 / 中转 ═════════►│   客户端 B    │
        │ (Wails + TUN) │                            │ (Wails + TUN) │
        └───────┬───────┘                            └───────┬───────┘
                │ 打洞（v4 与 v6 并行）                      │
                └══════════════ P2P 游戏数据 ═══════════════┘
                       （失败时回落到服务器中转）
                              ▲
                              │
                       ┌──────┴──────┐
                       │   服务端    │
                       │  （纯 Go）  │
                       └─────────────┘
```

## 🧱 技术栈

- **Go** 1.26
- **Wails** v2.12（桌面外壳）· **Vue 3** + **Vite** 3（前端）
- **wintun**（Windows 虚拟网卡）· **google/uuid**
- 服务端除共享的 `protocol` 模块外无任何第三方依赖。

## 🚀 快速开始

### 服务端

服务端是单个 Go 二进制——部署到一台开放了 UDP `10555` 的 VPS 上即可。

```bash
cd server
go run .                          # 默认监听 [::]:10555（双栈）

# 或构建后运行
go build -o netbridge-server .
./netbridge-server -addr 0.0.0.0:10555 -port 10555
```

命令行参数与环境变量（均可选）：

| 参数 | 环境变量 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `-addr` | — | `[::]:<port>` | 监听地址，如 `0.0.0.0:10555`（仅 v4） |
| `-port` | `NETBRIDGE_PORT` | `10555` | UDP 监听端口 |
| `-room-timeout` | `NETBRIDGE_ROOM_TIMEOUT` | `15` | peer 离线超时（秒） |

日志以结构化文本输出到 stdout，systemd 下由 journald 统一接管。

### 客户端（Windows）

需要安装 [Wails CLI](https://wails.io/docs/gettingstarted/installation) 与 Go。

```bash
cd client
wails dev      # 实时开发（热重载）
wails build    # 生成可分发的 .exe / NSIS 安装包
```

> 客户端创建 wintun 虚拟网卡需要**管理员权限**——Windows 清单已声明 `requireAdministrator`。在 `wails dev`（父进程非管理员）下有 `RestartAsAdmin` 兜底。

使用方式：启动应用 → 填服务端地址（`vps-ip:10555`）→ 用昵称加入房间 → 把房间号告诉朋友。服务端给你分配 VIP 后，虚拟网卡会自动开启。

## ⚙️ 工作原理（简述）

1. **加入**——客户端发送 `join_room`，服务端分配下一个空闲 VIP 并回 `room_status`（完整成员列表 + 你的 VIP）。
2. **发现**——服务端广播每个对端的公网 v4/v6 端点（`peer_address`），让所有人都能尝试打洞。
3. **打洞**——客户端并行经 v4、v6 交换打洞包，先建立的那条成为 P2P 通道。
4. **中转**——若 `PunchTimeout`（3 秒）内没有任何通道建立，数据被封装成紧凑 `Relay` 帧由服务端转发（服务端只按目的 VIP 路由，从不解析负载内容）。
5. **桥接**——TUN 桥从虚拟网卡读出裸 IP 包，解析目的 VIP，走 P2P（或中转）发出；入站包直接写回网卡。

关键常量（`protocol/constants.go`）：虚拟网段 `10.66.0.0/16`、MTU `1400`、心跳 `5s`、peer 超时 `15s`、打洞超时 `3s`、默认端口 `10555`。

## 📁 项目结构

```
NetBridge/
├── protocol/            # 共享线协议（编解码、数据帧、报文类型）
│   ├── frame.go         # 12 字节紧凑二进制数据帧
│   ├── packet.go        # JSON 信令报文信封 + 类型
│   └── constants.go     # 端口、VIP 前缀、MTU、超时
├── server/              # UDP 信令 + 中转服务端（纯 Go）
│   ├── internal/
│   │   ├── room/        # 房间与成员管理、VIP 分配
│   │   ├── relay/       # 中转路由（零拷贝帧转发）
│   │   ├── signaling/   # 加入/状态/退出/聊天处理
│   │   └── server/      # UDP 服务循环 + 分发
│   └── main.go
└── client/              # Wails 桌面客户端
    ├── app.go           # Wails 绑定（前端 ↔ 后端）
    ├── core/            # 客户端核心：连接、房间、聊天、通道
    ├── tun/             # wintun 适配器 + TUN↔网络桥接
    ├── netconn/         # UDP 连接、打洞
    ├── peer/            # 对端管理
    ├── frontend/        # Vue 3 + Vite 界面
    └── build/           # Windows 图标、NSIS 安装包、wintun.dll
```

## 📝 说明

- 客户端目前仅支持 Windows（wintun）。服务端可在任何能跑 Go 的平台上运行。
- 服务端必须开放 UDP `10555`，信令与中转都走它。
- P2P 能否成功取决于 NAT 类型；对称型 NAT 通常会回落到中转。
